HTML语法详细教程

1. HTML简介

HTML（HyperText Markup Language，超文本标记语言）是构建网页的标准标记语言。它使用标签来描述网页的结构和内容。

1.1 HTML文档基本结构

<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>页面标题</title>
</head>
<body>
    <!-- 页面内容 -->
</body>
</html>
  • <!DOCTYPE html>: 声明文档类型为HTML5
  • <html>: 根元素，包含整个网页
  • <head>: 包含元数据，不会在页面中显示
  • <body>: 包含页面的可见内容

2. 基本元素

2.1 标题标签

HTML提供六个级别的标题标签：

2.1.1 功能和用途

  • <h1>: 最高级别标题，通常用于页面主标题
  • <h2>: 二级标题，用于主要章节
  • <h3>: 三级标题，用于子章节
  • <h4>: 四级标题，用于小节
  • <h5>: 五级标题，用于更小的分组
  • <h6>: 最低级别标题，很少使用

2.1.2 语法结构

<h1>一级标题</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题</h6>

2.1.3 属性说明

  • id: 唯一标识符，可用于CSS样式和JavaScript操作
  • class: 类名，用于CSS样式
  • title: 提示文本，鼠标悬停时显示
<h1 id="main-title" class="page-title" title="这是主标题">网站首页</h1>
<h2 id="section1" class="section-title">第一章节</h2>

2.1.4 SEO和可访问性考虑

  • 每页应该只有一个 <h1> 标签
  • 标题层级应该连续，不要跳级（如从h1直接到h3）
  • 标题内容应该简洁明了，准确描述章节内容

2.2 段落标签

2.2.1 功能和用途

<p> 标签用于定义段落，是HTML中最基本的文本容器元素。浏览器会自动在段落前后添加空白间距。

2.2.2 语法结构

<p>这是一个段落。段落标签用于组织文本内容。</p>
<p>这是另一个段落。每个段落之间会有默认的间距。</p>

2.2.3 常用属性

<!-- 基本段落 -->
<p>普通段落内容</p>

<!-- 带CSS类的段落 -->
<p class="intro">引言段落</p>

<!-- 带ID的段落 -->
<p id="conclusion">结论段落</p>

<!-- 带样式的段落 -->
<p style="color: blue; font-size: 16px;">蓝色字体段落</p>

<!-- 带对齐方式的段落（HTML5中不推荐） -->
<p align="center">居中段落</p>

2.2.4 嵌套规则

  • 段落内可以包含行内元素（如 <span>, <a>, <strong> 等）
  • 段落内不能包含块级元素（如其他 <p>, <div>, <h1> 等）
<p>这是一个包含 <strong>加粗文本</strong> 和 <a href="#">链接</a> 的段落。</p>

2.3 链接标签

2.3.1 功能和用途

<a> 标签用于创建超链接，可以链接到其他网页、文件、邮箱、电话号码或页面内的特定位置。

2.3.2 核心属性详解

href 属性

href 是最重要的属性，指定链接的目标地址：

<!-- 绝对URL -->
<a href="https://www.example.com">外部网站</a>

<!-- 相对URL -->
<a href="about.html">关于页面</a>
<a href="../index.html">上级目录</a>
<a href="/contact.html">根目录</a>

<!-- 锚点链接 -->
<a href="#section1">跳转到本页章节</a>
<a href="page.html#section2">跳转到其他页面的章节</a>

<!-- 协议链接 -->
<a href="mailto:example@email.com">发送邮件</a>
<a href="tel:+8613800138000">拨打电话</a>
<a href="sms:+8613800138000">发送短信</a>

<!-- 文件下载 -->
<a href="document.pdf">下载PDF文件</a>
<a href="image.jpg">查看图片</a>
target 属性

控制链接的打开方式：

<!-- 当前窗口打开（默认） -->
<a href="page.html" target="_self">当前窗口</a>

<!-- 新窗口或新标签页打开 -->
<a href="https://www.example.com" target="_blank">新窗口打开</a>

<!-- 父框架打开 -->
<a href="page.html" target="_parent">父框架</a>

<!-- 顶层窗口打开 -->
<a href="page.html" target="_top">顶层窗口</a>

<!-- 指定框架打开 -->
<a href="page.html" target="frameName">指定框架</a>
其他重要属性
<!-- 下载属性 -->
<a href="file.pdf" download="新文件名.pdf">下载并重命名</a>

<!-- 关系属性 -->
<a href="https://external.com" rel="nofollow">不传递权重的外链</a>
<a href="https://external.com" rel="noopener">安全的外链</a>

<!-- 提示信息 -->
<a href="page.html" title="点击查看详细信息">链接文本</a>

<!-- ARIA标签（无障碍访问） -->
<a href="page.html" aria-label="查看产品详情页面">产品</a>

2.3.3 链接状态和伪类

CSS可以定义链接的不同状态：

<style>
a:link { color: blue; }      /* 未访问的链接 */
a:visited { color: purple; } /* 已访问的链接 */
a:hover { color: red; }      /* 鼠标悬停 */
a:active { color: orange; }  /* 鼠标按下 */
</style>

<a href="page.html">普通链接</a>

2.3.4 锚点链接详解

<!-- 创建锚点 -->
<h2 id="chapter1">第一章</h2>
<p id="paragraph1">这是第一段内容</p>

<!-- 链接到锚点 -->
<a href="#chapter1">跳转到第一章</a>
<a href="#paragraph1">跳转到第一段</a>

<!-- 平滑滚动（CSS） -->
<style>
html { scroll-behavior: smooth; }
</style>

2.3.5 安全性考虑

<!-- 外部链接安全写法 -->
<a href="https://external.com" target="_blank" rel="noopener noreferrer">
    安全的外部链接
</a>

2.4 图片标签

2.4.1 功能和用途

<img> 标签用于在网页中嵌入图片，是一个自闭合标签（不需要结束标签）。

2.4.2 核心属性详解

指定图片的源地址：

<!-- 相对路径 -->
<img src="images/photo.jpg" alt="照片">
<img src="../images/logo.png" alt="Logo">

<!-- 绝对路径 -->
<img src="/assets/images/banner.jpg" alt="横幅">

<!-- 完整URL -->
<img src="https://example.com/image.jpg" alt="外部图片">

<!-- Base64编码图片 -->
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mP8/5+hHgAHggJ/PchI7wAAAABJRU5ErkJggg==" alt="1x1像素图片">
alt 属性（必需）

替代文本，当图片无法显示时显示，也用于屏幕阅读器：

<img src="chart.jpg" alt="2025年销售数据图表显示增长了25%">
<img src="profile.jpg" alt="用户头像">
<img src="decoration.png" alt="">  <!-- 装饰性图片可以留空 -->
尺寸属性
<!-- 使用width和height属性 -->
<img src="photo.jpg" alt="照片" width="300" height="200">

<!-- 使用CSS样式（推荐） -->
<img src="photo.jpg" alt="照片" style="width: 300px; height: 200px;">

<!-- 响应式图片 -->
<img src="photo.jpg" alt="照片" style="max-width: 100%; height: auto;">

2.4.3 高级属性

<!-- 图片标题 -->
<img src="art.jpg" alt="艺术作品" title="鼠标悬停时显示的提示">

<!-- 图片加载方式 -->
<img src="photo.jpg" alt="照片" loading="lazy">     <!-- 懒加载 -->
<img src="photo.jpg" alt="照片" loading="eager">    <!-- 立即加载 -->

<!-- 图片解码方式 -->
<img src="photo.jpg" alt="照片" decoding="async">   <!-- 异步解码 -->
<img src="photo.jpg" alt="照片" decoding="sync">    <!-- 同步解码 -->

<!-- 跨域设置 -->
<img src="https://external.com/image.jpg" alt="外部图片" crossorigin="anonymous">

<!-- 图片映射 -->
<img src="map.jpg" alt="地图" usemap="#planetmap">
<map name="planetmap">
    <area shape="rect" coords="0,0,50,50" href="page1.html" alt="区域1">
    <area shape="circle" coords="100,100,50" href="page2.html" alt="区域2">
</map>

2.4.4 响应式图片

使用 srcset 属性
<!-- 不同分辨率的图片 -->
<img src="photo-400.jpg" 
     srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w"
     sizes="(max-width: 600px) 400px,
            (max-width: 1000px) 800px,
            1200px"
     alt="响应式照片">

<!-- 不同像素密度的图片 -->
<img src="photo.jpg"
     srcset="photo.jpg 1x,
             photo@2x.jpg 2x,
             photo@3x.jpg 3x"
     alt="高清图片">
使用 picture 元素
<picture>
    <!-- WebP格式（现代浏览器） -->
    <source srcset="image.webp" type="image/webp">
    <!-- AVIF格式（更新的浏览器） -->
    <source srcset="image.avif" type="image/avif">
    <!-- 默认JPEG格式 -->
    <img src="image.jpg" alt="图片描述">
</picture>

<!-- 基于媒体查询的响应式图片 -->
<picture>
    <source media="(min-width: 800px)" srcset="large.jpg">
    <source media="(min-width: 400px)" srcset="medium.jpg">
    <img src="small.jpg" alt="响应式图片">
</picture>

2.4.5 图片优化最佳实践

<!-- 1. 提供适当的alt文本 -->
<img src="graph.png" alt="2024年第四季度销售额比第三季度增长15%">

<!-- 2. 使用合适的图片格式 -->
<picture>
    <source srcset="photo.webp" type="image/webp">  <!-- 现代格式 -->
    <source srcset="photo.jpg" type="image/jpeg">   <!-- 回退格式 -->
    <img src="photo.jpg" alt="照片">
</picture>

<!-- 3. 实现懒加载 -->
<img src="placeholder.jpg" 
     data-src="actual-image.jpg" 
     loading="lazy" 
     alt="描述">

<!-- 4. 设置合适的尺寸 -->
<img src="photo.jpg" 
     alt="照片" 
     width="300" 
     height="200"
     style="object-fit: cover;">

2.4.6 常见错误和解决方案

<!-- ❌ 错误：缺少alt属性 -->
<img src="photo.jpg">

<!-- ✅ 正确：包含alt属性 -->
<img src="photo.jpg" alt="用户头像">

<!-- ❌ 错误：使用图片作为文本 -->
<img src="text-image.jpg" alt="点击这里">

<!-- ✅ 正确：使用真实文本 -->
<button>点击这里</button>

<!-- ❌ 错误：不合适的文件格式 -->
<img src="large-photo.bmp" alt="照片">

<!-- ✅ 正确：使用web优化格式 -->
<img src="large-photo.webp" alt="照片">

2.5 列表

2.5.1 无序列表 (<ul>)

功能和用途

无序列表用于显示没有特定顺序的项目集合，默认使用圆点作为项目符号。

语法结构
<ul>
    <li>第一项</li>
    <li>第二项</li>
    <li>第三项</li>
</ul>
属性和样式
<!-- 自定义项目符号类型（HTML5中已废弃，推荐使用CSS） -->
<ul style="list-style-type: disc;">    <!-- 实心圆点（默认） -->
    <li>项目1</li>
</ul>

<ul style="list-style-type: circle;">  <!-- 空心圆点 -->
    <li>项目1</li>
</ul>

<ul style="list-style-type: square;">  <!-- 实心方块 -->
    <li>项目1</li>
</ul>

<ul style="list-style-type: none;">    <!-- 无符号 -->
    <li>项目1</li>
</ul>

<!-- 嵌套列表 -->
<ul>
    <li>主项目1
        <ul>
            <li>子项目1.1</li>
            <li>子项目1.2</li>
        </ul>
    </li>
    <li>主项目2</li>
</ul>

2.5.2 有序列表 (<ol>)

功能和用途

有序列表用于显示有特定顺序或优先级的项目集合，使用数字、字母或罗马数字作为项目符号。

语法结构和属性
<!-- 基本有序列表 -->
<ol>
    <li>第一步</li>
    <li>第二步</li>
    <li>第三步</li>
</ol>

<!-- 指定起始数字 -->
<ol start="5">
    <li>第五项</li>    <!-- 显示为 5. -->
    <li>第六项</li>    <!-- 显示为 6. -->
</ol>

<!-- 编号类型 -->
<ol type="1">    <!-- 数字（默认）：1, 2, 3 -->
    <li>项目</li>
</ol>

<ol type="A">    <!-- 大写字母：A, B, C -->
    <li>项目</li>
</ol>

<ol type="a">    <!-- 小写字母：a, b, c -->
    <li>项目</li>
</ol>

<ol type="I">    <!-- 大写罗马数字：I, II, III -->
    <li>项目</li>
</ol>

<ol type="i">    <!-- 小写罗马数字：i, ii, iii -->
    <li>项目</li>
</ol>

<!-- 反向编号 -->
<ol reversed>
    <li>最后一项</li>    <!-- 显示为 3. -->
    <li>中间项</li>      <!-- 显示为 2. -->
    <li>第一项</li>      <!-- 显示为 1. -->
</ol>

<!-- 嵌套有序列表 -->
<ol>
    <li>第一章
        <ol type="A">
            <li>第一节</li>
            <li>第二节</li>
        </ol>
    </li>
    <li>第二章</li>
</ol>
列表项属性
<ol>
    <li value="10">这一项显示为10</li>  <!-- 指定特定值 -->
    <li>这一项显示为11</li>            <!-- 继续递增 -->
    <li value="20">这一项显示为20</li>  <!-- 重新指定值 -->
    <li>这一项显示为21</li>            <!-- 继续递增 -->
</ol>

3. 文本格式化

3.1 强调和重要性标签

3.1.1 语义化强调标签

<em> 标签 - 强调
<!-- 用于强调文本，通常显示为斜体 -->
<p>这件事<em>非常</em>重要。</p>
<p>请<em>仔细</em>阅读说明书。</p>

<!-- 可以嵌套使用增强强调 -->
<p>这是<em>重要的，<em>非常</em>重要</em>。</p>
<strong> 标签 - 重要性
<!-- 用于表示重要性，通常显示为粗体 -->
<p><strong>警告：</strong>此操作不可撤销。</p>
<p><strong>注意：</strong>保存前请检查所有信息。</p>

<!-- 可以嵌套使用增强重要性 -->
<p><strong>这很重要，<strong>极其</strong>重要</strong>。</p>

3.1.2 视觉样式标签（不推荐用于语义）

<!-- 斜体样式（仅视觉效果，无语义含义） -->
<i>斜体文本</i>
<i class="icon-star"></i>  <!-- 常用于图标 -->

<!-- 粗体样式（仅视觉效果，无语义含义） -->
<b>粗体文本</b>
<b class="keyword">关键词</b>  <!-- 用于关键词高亮 -->

3.1.3 语义 vs 样式对比

<!-- ✅ 推荐：语义化标签 -->
<p>请<strong>立即</strong>处理这个<em>紧急</em>任务。</p>

<!-- ❌ 不推荐：仅为样式使用 -->
<p>请<b>立即</b>处理这个<i>紧急</i>任务。</p>

<!-- ✅ 正确：用CSS控制样式 -->
<style>
em { font-style: normal; color: red; }

5.1 基本表单结构详解

5.1.1 <form> 标签 - 表单容器

功能说明

<form> 标签用于创建用户输入表单，它是所有表单元素的容器，负责收集用户数据并将其发送到服务器进行处理。

语法结构和示例
<form action="目标URL" method="提交方法" enctype="编码类型">
    <!-- 表单元素 -->
</form>
重要属性详解

action 属性

<!-- 指定表单数据提交的目标URL -->
<form action="/submit-form">
    <!-- 表单内容 -->
</form>

<!-- 提交到当前页面 -->
<form action="">
    <!-- 表单内容 -->
</form>

<!-- 提交到外部服务 -->
<form action="https://example.com/api/contact">
    <!-- 表单内容 -->
</form>

method 属性

<!-- GET方法：数据附加到URL中，适合查询 -->
<form action="/search" method="get">
    <input type="search" name="q" placeholder="搜索内容">
    <button type="submit">搜索</button>
</form>

<!-- POST方法：数据在请求体中，适合敏感数据 -->
<form action="/login" method="post">
    <input type="email" name="email" required>
    <input type="password" name="password" required>
    <button type="submit">登录</button>
</form>

enctype 属性

<!-- 默认编码：application/x-www-form-urlencoded -->
<form action="/submit" method="post">
    <input type="text" name="username">
</form>

<!-- 文件上传编码：multipart/form-data -->
<form action="/upload" method="post" enctype="multipart/form-data">
    <input type="file" name="avatar" accept="image/*">
    <button type="submit">上传</button>
</form>

<!-- 纯文本编码：text/plain（很少使用） -->
<form action="/feedback" method="post" enctype="text/plain">
    <textarea name="message"></textarea>
</form>

5.1.2 <label> 标签 - 表单标签

功能说明

<label> 标签为表单元素提供可点击的标签，提高用户体验和可访问性。点击标签会自动聚焦到关联的表单元素。

语法结构和示例
<!-- 方法1：使用for属性关联 -->
<label for="element-id">标签文本</label>
<input type="text" id="element-id" name="field-name">

<!-- 方法2：嵌套关联 -->
<label>
    标签文本
    <input type="text" name="field-name">
</label>
详细示例
<!-- 文本输入字段 -->

       id="username" 
       name="username" 
       required>

<input type="password" 
       id="password" 
       name="password" 
       required>

<!-- 嵌套样式的标签 -->

    <span>邮箱地址：</span>
    <input type="email" name="email" placeholder="请输入邮箱">
</label>

<!-- 单选按钮标签 -->

11. HTML验证

HTML验证是确保代码质量和标准兼容性的重要步骤。

11.1 为什么需要验证

11.1.1 验证的重要性

  • 标准兼容性：确保代码符合W3C标准
  • 跨浏览器兼容：减少浏览器兼容性问题
  • 可维护性：提高代码的可读性和维护性
  • SEO优化：搜索引擎更好地理解页面结构
  • 无障碍访问：确保所有用户都能访问内容

11.2 常见HTML错误

11.2.1 结构错误

<!-- 错误：未闭合标签 -->
<div>
    <p>段落内容
    <span>文本内容
</div>

<!-- 正确：所有标签都正确闭合 -->
<div>
    <p>段落内容</p>
    <span>文本内容</span>
</div>

11.2.2 属性错误

<!-- 错误：重复的ID -->
<div id="content">内容1</div>
<div id="content">内容2</div> <!-- ID不能重复 -->

<!-- 错误：属性值未加引号 -->
<img src=image.jpg alt=图片描述>

<!-- 正确写法 -->
<div id="content1">内容1</div>
<div id="content2">内容2</div>
<img src="image.jpg" alt="图片描述">

11.2.3 嵌套错误

<!-- 错误：块级元素嵌套在内联元素中 -->
<span>
    <div>这是错误的嵌套</div>
</span>

<!-- 错误：交叉嵌套 -->
<strong><em>文本内容</strong></em>

<!-- 正确写法 -->
<div>
    <span>这是正确的嵌套</span>
</div>

<strong><em>文本内容</em></strong>
