Markdown 语法详解

本文完全由Claude Sonnet 4 生成，仅作为一个快速查阅的手册和参考使用。

1. 标题（Headings）

Markdown 语法： 

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

对应 HTML： 

<h1>一级标题</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题</h6>

2. 段落与换行

段落由一个或多个空行分隔。要强制换行，可在行末添加两个空格或使用 <br>。

Markdown： 

第一段落内容。

第二段落内容。

这是第一行内容··  
这是第二行内容

HTML： 

<p>第一段落内容。</p>
<p>第二段落内容。</p>
<p>这是第一行内容<br>
这是第二行内容</p>

渲染效果：

第一段落内容。

第二段落内容。

这是第一行内容
这是第二行内容

3. 强调与加粗

Markdown HTML 渲染效果
*斜体* 或 _斜体_ <em>斜体</em> 斜体
**加粗** 或 __加粗__ <strong>加粗</strong> 加粗
~~删除线~~ <del>删除线</del> 删除线

4. 列表（Lists）

4.1 无序列表

Markdown： 

- 苹果
- 香蕉
  - 脐橙
  - 血橙
* 或者星号
+ 或者加号

HTML： 

<ul>
  <li>苹果</li>
  <li>香蕉
    <ul>
      <li>脐橙</li>
      <li>血橙</li>
    </ul>
  </li>
  <li>或者星号</li>
  <li>或者加号</li>
</ul>

渲染效果：

  • 苹果
  • 香蕉
    • 脐橙
    • 血橙
  • 或者星号
  • 或者加号

4.2 有序列表

Markdown： 

1. 第一步
2. 第二步
3. 第三步
   1. 子步骤 a
   2. 子步骤 b

HTML： 

<ol>
  <li>第一步</li>
  <li>第二步</li>
  <li>第三步
    <ol>
      <li>子步骤 a</li>
      <li>子步骤 b</li>
    </ol>
  </li>
</ol>

渲染效果：

  1. 第一步
  2. 第二步
  3. 第三步
    1. 子步骤 a
    2. 子步骤 b

5. 链接（Links）

5.1 行内链接

Markdown： 

[Google](https://www.google.com "谷歌搜索")

HTML： 

<a href="https://www.google.com" title="谷歌搜索">Google</a>

渲染效果：
Google

Markdown： 

访问 [百度][1] 或 [GitHub][].

[1]: https://www.baidu.com "百度搜索"
[GitHub]: https://github.com

渲染效果：
访问 百度 或 GitHub

24. Markdown最佳实践

24.1 文档结构建议

  1. 使用有层次的标题

    • 一个文档只使用一个 # 标题
    • 标题层级不要跳跃（不要从 # 直接跳到 ###）
    • 每个标题前后都要有空行

  2. 合理使用空行

    • 段落之间用空行分隔
    • 列表前后要有空行
    • 代码块前后要有空行

  3. 链接处理

    • 对于重复使用的链接，使用引用式链接
    • 外部链接考虑添加 target="_blank"

24.2 编写技巧

  1. 表格设计

    • 保持表格简洁，避免过多列
    • 使用对齐方式提高可读性
    • 复杂数据考虑分割成多个表格

  2. 代码块优化

    • 总是指定编程语言
    • 添加注释说明关键代码
    • 保持代码的可读性和简洁性

  3. 图片处理

    • 使用有意义的 alt 文本
    • 控制图片大小和格式
    • 考虑使用图片说明文字
